# ZCU104 AI Accelerator - Runtime Configuration
   
   ## Environment
   - **Board**: ZCU104 Rev 1.0
   - **Vivado/Vitis**: 2024.2
   - **OS**: Bare-metal (standalone)
   - **Processor**: Cortex-A53 (psu_cortexa53_0)
   - **Clock Frequency**: 1.2 GHz (default)
   
   ## Build Configuration
   - **Optimization**: -O2
   - **Debug**: Enabled
   - **Compiler**: ARM GCC

   ### 1. -O2의 의미
   - **컴파일러의 개입**: 코드를 그대로 기계어로 바꾸는 것이 아니라, 컴파일러가 논리적으로 분석하여 중복된 계산을 합치고, 실행 순서를 조절하여 CPU가 더 빠르게 처리할 수 있게 코드를 재구성합니다.
   - **균형점(Standard)**: 실행 속도와 코드 크기 사이에서 가장 적절한 균형을 맞춘 단계입니다. 임베디드 시스템(A53 코어 등)에서 실무적으로 가장 많이 사용되는 표준 옵션입니다.

   ### 2. 주요 최적화 레벨 비교
   | 레벨 | 특징 | 주요 용도 및 장단점 |
   | :---: | :--- | :--- |
   | **`-O0`** | **최적화 없음** | **디버깅 전용.** 코드와 기계어가 1:1 매칭되어 중단점(Breakpoint) 추적이 정확합니다. |
   | **`-O1`** | **기본 최적화** | **기초 성능 향상.** 컴파일 시간과 실행 성능의 최소한의 균형을 맞춥니다. |
   | **`-O2`** | **중상급 최적화** | **실전/배포용.** 속도가 크게 개선되며, 임베디드 프로젝트의 가장 표준적인 옵션입니다. (코드 점프 현상 발생 가능) |
   | **`-O3`** | **공격적 최적화** | **고성능 연산.** 루프 최적화 등을 통해 속도를 극대화하지만 코드 크기가 커지고 예기치 못한 버그가 생길 수 있습니다. |
   | **`-Os`** | **크기 최적화** | **메모리 절약.** 속도보다 실행 파일의 크기를 줄이는 데 집중합니다. 메모리가 극히 제한된 환경에서 사용합니다. |
   | **`-Ofast`** | **제한 없는 최적화** | **극한의 속도.** 표준 규격을 무시하더라도 속도만을 위해 모든 기법을 동원합니다. 정밀도가 중요한 연산엔 위험할 수 있습니다. |
   
   ## Boot Mode
   - **Mode**: JTAG
   - **SW6 Setting**: 0x0008
   
   ## UART Settings
   - **Baud Rate**: 115200
   - **Data Bits**: 8
   - **Parity**: None
   - **Stop Bits**: 1
